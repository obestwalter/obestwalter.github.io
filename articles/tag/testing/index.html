    3 dimensional testing
    • 3 sizes (micro, feature, vertical)
    • 3 speeds (fast, medium, slow) (or 2)
    • 3 stages to run on (dev, test, acc)
    • 3 setups (roles? types? kinds?) to run on (dev, CI, deployed)

    Sometimes you have less dimensions but in most cases I would say you don’t need more some of them are ranges where we need to make informed decisions about arbitrary cut-off points (size, speed), some of them are distinct (stages, setups)
